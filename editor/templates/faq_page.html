Security

How is my communication protected?

NoTex applies standard SSL with 256 bit encryption, based on a certificated issued by GoDaddy.com. That means nobody between your computer and the editor's server can eavesdrop and access the data. You can confirm this statement by clicking on the padlock icon in the address bar of your browser. Plus you should see the SSL seal at the footer of every page: If you click on it you'll see the issuer's verification.

How is my data protected?

Your data is kept on servers in the European Union; the data itself is not encrypted, which means the administrators could have access to it, although internal policy forbids this. Having said that, it's planned to introduce client side encryption, in which case even a person with physical access to the hard disks would not be able to read your data.

If you do not feel comfortable with the current situation, then there is an offline solution available at the download page: Once installed, it works even without an internet access.

Where do I sign up & login?

NoTex does not require you to sign up or login; just use it. If you re-visit the site then you're identified based on a secure cookie which has been generated and stored (in your browser) at your last visit. If you delete your cookie then there is no possibility to associate you with the last visit; therefore the editor will create a new session for you.

Data

How much space do I have?

Not much; NoTex is not meant as a storage solution for large amounts of data. But it is planned to introduce a Dropbox integration, where your data within the editor would be stored (encrypted!) on Dropbox' servers. At the moment it's recommended, to export regularly your data to your computer. Or you get the offline solution from the download page, which works locally.

How long do you keep my data?

If you do not re-visit NoTex for longer than 3 days, then all your data is deleted. It is not recoverable and there is no support for any requests w.r.t. lost data. As long as you keep using NoTex on a regular basis (at least once every 72 hours), then your data is kept on the servers.

I deleted my cookies; can I access my data?

No; your data will automatically be deleted within 3 days. NoTex uses a secure cookie to identify re-visiting users; if you delete it, then there is no possibility to associate you with your now in-accessible data. Either export regularly your data to your computer, and re-import it in such a situation, or use the offline solution available at the download page.

Performance

Why does the PDF conversion take so long?

Converting reStructuredText to LaTex is not an expensive operation, but translating LaTex to PDF involves significant amount of processing; even on modern machines it can take not only milli-seconds but some seconds and even minutes.

Can the PDF conversion be accelerated?

Yes, to some extend: NoTex uses by default the xelatex backend to do the LaTex translation, which is compared to the pdflatex backend slower (not terribly but significantly). The wiki explains how you can change the backend.

It can also happen, that our servers cannot cope with the user demand: It such cases either simply wait and re-try later on, or check out the offline solution; see download page.

The options.cfg wiki page explains in detail how to use the configuration file and apply it within a report or project. It basically contains meta-information about the project and allows tweaking of the conversion parameters.

Miscellanea

What does NoTex mean?

It's a pun on Tex, since if the NoTex editor is properly used, you can avoid using (and learning) Tex or LaTex. Knowing LaTex is not a disadvantage though, since you can apply it to change NoTex' default templates.

Is it open source?

Yes it is; the source code is available at the GIT repository http://github.com/hsk81/notex and is GPL licenced. Execute git clone git://github.com/hsk81/notex.git to get a copy.

What technology do you use?

The editor's front-end has entirely been written with the ExtJS JavaScript framework plus the JavaScript code editor component CodeMirror; and for the back-end the Django Python web framework is being used. The back-end applies the Python document generator Sphinx to translate the RST input files to the various target formats; reStructuredText is the "markup syntax and parser component" of Docutils. LaTex is translated to PDF using TeX Live.

There is some work underway to split the project's editor and translation part using the Python micro-framework Flask and the Python distributed task queue Celery. But this will take some time; see WebEd.
