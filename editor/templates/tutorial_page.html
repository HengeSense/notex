This tutorial helps you to familiarize yourself with the user interface of NoTex and guides you through your first reStructuredText project. The first section is about the user interface and explains the corresponding functionality. The impatient reader can skip it and continue directly with the first project section.

User Interface

The user interface is split into four areas: On the left the report manager (with its own toolbar), in the central area either the NoTex menu (including this tutorial page) or the editor (not visible yet) and in the header the main toolbar on top of the central area; the footer contains the status bar.

Report Manager

On the left hand side, you should see the report manager; if you don't, then it is possibly hidden and you have to un-hide it by clicking the small arrow button in the middle of the vertical separator. The same arrow button serves also to hide the report manager. The report (aka project) manager allows you to group various files into projects.

Toolbar

The report manager's toolbar allows you to create, read, update and delete the reports. You can import and export them and you can open local files to be imported into the currently selected report.

Editor

In the central area where you read this tutorial, you'll be editing your files inside an editor. It is not visible at the moment, since you've not opened any file yet. So to check out the editor, first open NoTex in a second tab by clicking the following link:

  • NoTex Editor.

Continue applying the instructions in the second tab, while reading the tutorial in the first one. Now, click in the manager on the small plus sign of e.g. Complex Article; you can also double click the report icon or title: It will fetch the required data and display a list of associated files.

They come in three kinds: CFG, TXT and JPG. The option.cfg file is a configuration describing what kind of project you're working with. It is generated by a wizard, which we'll soon have a look at; the content.txt file contains the actual corpus and the other files are JPEG images.

Click on content.txt to open it. The editor will cover the central area, which means that you've to continue reading this tutorial in the first tab. You should now be looking at a Wikipedia article about mass–energy equivalence, which has been for demonstration purposed copied and reformatted as reStructuredText.

Toolbar

The editor supports syntax highlighting and many of the standard interactions you're probably familiar with from other text processors. But this one has a toolbar which specifically supports reStructuredText. That means, if e.g. you don't know how to strongly emphasize a couple of words with the reStructuredText markup, then you can simply select and mark the words and click the icon with the capital "B"; the words will be surrounded with asterisks which is the restructured way of telling that you want to strongly emphasize those words.

Do it: Go ahead and select e.g. the first occurrence of mass–energy equivalence and press the strong emphasis button. The result should be something like **mass–energy equivalence**. Using the editor's toolbar you can learn the reStructuredText markup interactively since the meanings of all the icons on the toolbar are generally known from other applications or are self-descriptive: If you hover over a button you should see a tip box appearing.

Close the editor by clicking in the footer the "x" button next to the title of the current document content.txt.

Main Toolbar

The main toolbar offers the same functions as the report manager's toolbar, but each icon has a description label next to it.

Status Bar

The status bar shows what NoTex is doing at the moment and it allows you also to set the general font size of the editor; if there is not editor open yet, then you won't see any reaction. Otherwise, it also displays the cursor's current line and position; if you click on that information, then the number of lines, words and characters is shown.

First Project

In the main toolbar click on the small arrow next to the Add button, and select from the drop down menu the Report entry. The Create Report wizard should appear: Enter Tutorial for the project name and your own name for author(s). Skip all the other values, but the last one: Select for content tutorial (instead of empty) and click the Create button.

A new Tutorial project should be created. In the report manager open the corresponding content.txt file: You should see now an example reStructuredText markup explaining .. well reStructuredText! You can read it later.

Now, just hit the Export as .. PDF button, and NoTex should deliver in a few seconds a PDF version for the current project; congratulations! You can also experiment with exporting as LaTex or HTML. Or you could do a simple export and re-import the project back into NoTex.

What now?

Read the just exported PDF (or here as HTML) to learn more about reStructuredText; check out also the other example projects and experiment with the various export possibilities. There is also a wiki where you can learn more about the option.cfg and content.txt files plus ask questions and provide feedback.
