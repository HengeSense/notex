NoTex is a browser based editor, which allows you to write reports and theses based on the reStructuredText markup language. Once written the documents can be exported as PDF, HTML or LaTex.

If you feel ‒ for various reasons ‒ uncomfortable using an online editor, then you may be interested in an offline solution, which can be used with or even without an internet connection.

Download

The offline editor is packaged as a so called virtual machine for VirtualBox; based on your needs, you can choose from one of the following two options:
  • 1. NoTex International

Offers full support for the LaTex backend Texlive 2012, whereas the far eastern Chinese, Japanese and Korean languages are included. Plus it contains fonts for various other Asian languages. A list of supported packages can be found here.
  • 2. NoTex non-CJK

This non-CJK edition lacks support for Chinese, Japanese and Korean; plus many Asian fonts have been omitted. The list of supported and omitted packages can be found here.

In addition to one of the above virtual machines, VirtualBox itself is required to run the system; it's available at:

  • https://www.virtualbox.org.

Go to the corresponding download section, and choose the version compatible with your operating system. Once VirtualBox is properly setup, you may continue with the installation of NoTex.

Installation

After you've setup VirtualBox, you have to import the international or the non-CJK version; from the menu of the main window choose:

  • » File » Import Appliance ...

and follow the instructions of the import wizard. Once the import is done, start the virtual machine.

VM Import Wizard: It is possible to tweak the virtual machine settings at import, e.g. you can increase the number of CPUs or the amount of RAM allocated to experience better performance (the defaults should be more than enough though). Deselecting the DVD check box and reinitializing the MAC address are recommended.

Usage

If not already done, start the NoTex virtual machine: As soon as it is running, point your browser of choice (not Internet Explorer) to the address:

  • http://localhost:8080/.

If you run the editor on a different host than the browser, then you may have to replace localhost with the correct host name.

NoTex VM: Once the virtual machine has booted up, you should see a screen similar to what is shown above. Let the VM run in the background, open a browser and enter the address http://localhost:8080/ to connect to the application.

Although the virtual machine shows a login prompt at the console, it is not possible to do so, since for security reasons all password logins to the system have been disabled. Your data is stored within the VM, so make sure to export your data (using NoTex) in case you intend to delete the VM completely.

Should your mouse pointer get captured by the VM, then press the host key (usually the right CTRL key, see the lower right corner of the status bar) to get it released.
